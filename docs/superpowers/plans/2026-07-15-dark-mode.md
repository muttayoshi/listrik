# Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a fully-covered dark mode to Listrikku (dashboard + all modals/forms) with a Sistem/Terang/Gelap toggle in Settings, persisted independently of the app's Dexie-backed data.

**Architecture:** Tailwind v4 class-based dark strategy (`@custom-variant dark (&:where(.dark, .dark *));`) toggling a `.dark` class on `<html>`. A small `src/theme.ts` module owns the preference (localStorage, key `listrikku-theme`) and exposes a `useTheme()` hook. Every component gets `dark:` variants added directly next to its existing Tailwind classes (no CSS-variable/token refactor), following the mapping table in the approved spec.

**Tech Stack:** React 19, Vite, TypeScript, Tailwind CSS v4 (no PostCSS config, no Tailwind config file — `@theme`/`@custom-variant` live in `src/index.css`).

**Spec:** `docs/superpowers/specs/2026-07-15-dark-mode-design.md` — read it once before starting; this plan implements it exactly.

## Global Constraints

- Storage key is exactly `listrikku-theme` (localStorage), values are exactly `'system' | 'light' | 'dark'` — this string appears hardcoded in both `index.html`'s inline script and `src/theme.ts`; keep them byte-for-byte identical.
- Tailwind dark strategy is class-based via `@custom-variant dark (&:where(.dark, .dark *));` added to `src/index.css` — do not rely on the Tailwind v4 default media-query-only `dark:` behavior.
- Theme preference must **never** be added to Dexie `Settings`, `ExportPayload`, or the share-link payload. Do not touch `src/db.ts`, `src/exportImport.ts`, or `src/share.ts` in this plan.
- Card/border/hover dark tones use the **emerald-950/900/800** family (not neutral `gray-900/800`) to match the existing `--color-canvas: #0d2a1f` page background. Text stays neutral (`gray-100/300/400/500/600`).
- **Unchanged in dark mode (by design, not an oversight):** every inline `linear-gradient(135deg, #059669, #047857, ...)` brand gradient (header logo, FABs, all "save"/"share"/"import" buttons, the big `SummaryCards`/`ShareView` cost card), the `DeviceCard` cost-intensity `hsl(...)` bar/dot, the `bg-black/40 backdrop-blur-sm` modal overlay, and the light-green empty-state icon gradient in `App.tsx`.
- This repo has **no automated test framework** (no vitest/jest, confirmed via `package.json`/`tsconfig.json` — no test script, no test libs). "Testing" in every task below means: `pnpm build` (TypeScript/build correctness) + manual visual check in the already-running dev server (per `AGENTS.md`, a Vite dev server is always up on `$PORT`).
- Every task's className edits are additive — you're adding `dark:...` utilities next to existing classes, never removing or renaming an existing light-mode class.

- [ ] **Step 1: Create `src/theme.ts`**

```typescript
import { useEffect, useState } from 'react'

export type ThemePreference = 'system' | 'light' | 'dark'

const STORAGE_KEY = 'listrikku-theme'

export function getStoredTheme(): ThemePreference {
  try {
    const stored = localStorage.getItem(STORAGE_KEY)
    if (stored === 'light' || stored === 'dark' || stored === 'system') return stored
  } catch {
    // localStorage unavailable (e.g. Safari private mode) — fall back to 'system'
  }
  return 'system'
}

function storeTheme(theme: ThemePreference) {
  try {
    localStorage.setItem(STORAGE_KEY, theme)
  } catch {
    // localStorage unavailable — preference only persists for this session
  }
}

function prefersDark(): boolean {
  return typeof window.matchMedia === 'function' && window.matchMedia('(prefers-color-scheme: dark)').matches
}

export function resolveEffectiveTheme(theme: ThemePreference): 'light' | 'dark' {
  if (theme === 'system') return prefersDark() ? 'dark' : 'light'
  return theme
}

export function applyTheme(theme: ThemePreference) {
  const effective = resolveEffectiveTheme(theme)
  document.documentElement.classList.toggle('dark', effective === 'dark')
}

export function useTheme(): [ThemePreference, (t: ThemePreference) => void] {
  const [theme, setThemeState] = useState<ThemePreference>(() => getStoredTheme())

  useEffect(() => {
    applyTheme(theme)
  }, [theme])

  useEffect(() => {
    if (theme !== 'system' || typeof window.matchMedia !== 'function') return
    const mq = window.matchMedia('(prefers-color-scheme: dark)')
    function handleChange() {
      applyTheme('system')
    }
    mq.addEventListener('change', handleChange)
    return () => mq.removeEventListener('change', handleChange)
  }, [theme])

  function setTheme(next: ThemePreference) {
    storeTheme(next)
    setThemeState(next)
  }

  return [theme, setTheme]
}
```

Note: `STORAGE_KEY` must stay exactly `'listrikku-theme'`, matching the string hardcoded in `index.html`'s inline script from Task 1.
